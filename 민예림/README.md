### 💜 240909 (월)

#### 💡 작업 상황
- Widerface 데이터 라벨링, 전처리 완료

- 그래서 colab에서 학습 시키려고 dataset을 드라이브로 옮기는 중인데 이것도 너무 많아서 오래걸리는 중...
- 지난주에 테스트한 내용들 정리함.


#### 💡 이슈
- 데이터 라벨링 잘 끝냈는데, YOLO에서 원하는 형식은 class_id가 포함되어 있어야 해서 계속 5개의 컬럼을 요구하는데, 4개 밖에 없다는 오류가 떴다.<br>
    => 라벨링한 모든 txt 파일의 맨 앞 부분에 얼굴을 가리키는 class_id인 '0'을 붙여준다.
    자동화 코드로 진행
- YOLO의 바운딩 박스 좌표는 0-1 사이의 값으로 정규화되어야 한다. 형식에 맞도록 변환하는 코드가 또 필요했고, 음수값이 나오거나, 이미지의 범위를 넘어갔을 경우도 고려한 코드가 필요했다.<BR>
    => 정규화 함수를 적용하여 좌표를 변환해서 저장해주도록 했고, 음수 값이 나오거나 이미지의 범위를 넘어간 경우 0으로 처리하도록 했다.<BR>
    => 이게 맞는 지는 학습 완료되고 테스트 해봐야 알 것 같다.


#### 💡 오늘의 느낌
- conda 명령어 다 외우겠다.. 오늘 Jupyter Hub 안되서 몇 번을 반복해서 이제 안보고도 친다.. 내일은 꼭 오류 해결할 수 있길..ㅜㅜ
- 뭔가 될듯 말듯 안되네.. 금방 될 것 같은데 학습이 또 오래 걸리고 ㅜㅜ GPU 서버 세팅만 잘 되면 그래도 학습도 조금 빠르게 완료될 것 같은데ㅠㅠ
